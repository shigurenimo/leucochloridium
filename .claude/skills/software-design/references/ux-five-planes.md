# UX 5段階モデル

Jesse James Garrett『Elements of User Experience』に基づく。下層が上層の土台になる。戦略がブレると全層がブレる。段階を行き来しながら進めるが、今どの段階かは常に意識する。

## 段階

- 戦略 (Strategy): ユーザーは何を求め、事業は何を達成するか
- 要件 (Scope): どの機能・コンテンツが必要か
- 構造 (Structure): 情報をどう整理し、どう遷移させるか
- 骨格 (Skeleton): 画面のどこに何を配置するか
- 表層 (Surface): 色、タイポグラフィ、ビジュアル

## .docs との対応

- 戦略 ⇒ `index.md`（ジョブ、ビジョン、原則）
- 要件 ⇒ `backlogs/`（課題分析と対応案）
- 構造 ⇒ `user-flows.md`、`domain-model.md`
- 骨格 ⇒ `features.md`、`sitemap.md`
- 表層 ⇒ コードとデザインツールが仕様
